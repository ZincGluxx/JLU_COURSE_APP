# 🎯 课表数据提取 - 增强版使用说明

## ✨ 新功能特性

### 1. **智能等待和重试机制**
- 自动检测页面是否为动态加载
- 如果页面内容为空，自动等待并重试（最多3次）
- 每次重试间隔3秒

### 2. **多种提取方法**
支持3种数据提取方式：

#### 方法1：`.kbcontent` 元素（最常见）
```javascript
document.querySelectorAll('.kbcontent')
```
适用于大多数教务系统的课表页面

#### 方法2：`table` 表格解析
```javascript
document.querySelectorAll('table[id*="kb"]')
```
适用于使用表格布局的课表

#### 方法3：JavaScript 全局变量（新增）
```javascript
window.kbData, window.courseData, window.scheduleData
```
从页面的JavaScript变量中直接读取数据

### 3. **增强的页面诊断**
显示详细的调试信息：
- ✅ 页面加载状态 (`readyState`)
- ✅ 找到的元素数量
- ✅ 页面是否为空（动态加载检测）
- ✅ 使用的提取方法
- ✅ jQuery 可用性检测

## 📱 使用步骤

### 方式A：在应用中使用（推荐）

```
1. 打开应用 → 设置 → 重新获取课表
2. 登录 i.jlu.edu.cn
3. 进入"我的课表"页面
4. 等待课表完全显示
5. 点击 📥 提取按钮
```

**智能提取过程：**
- ⏳ 显示"正在提取课表数据..."
- 🔄 自动等待2秒让页面加载
- 🔍 开始提取，如果失败会自动重试
- ✅ 成功后显示课程数量并返回

### 方式B：JSON导入（最可靠）

如果应用内提取失败：
```
1. 电脑浏览器打开课表页面
2. F12 → Console
3. 运行提取脚本（见 JSON_IMPORT_GUIDE.md）
4. 应用 → 设置 → 导入课表JSON
5. 粘贴并导入
```

## 🐛 故障排查

### 问题：提取时提示"页面内容为空"

**原因**：
- 页面是动态加载的
- JavaScript还在执行中
- 没有连接到服务器

**解决**：
1. ✅ **等待更长时间**
   - 应用会自动重试3次
   - 每次等待3秒
   
2. ✅ **使用在线页面**
   - 不要用本地HTML文件
   - 访问 https://iedu.jlu.edu.cn
   
3. ✅ **切换到JSON导入**
   - 更可靠的方式
   - 查看 JSON_IMPORT_GUIDE.md

### 问题：找到0个课表元素

**检查清单**：
- [ ] 是否在课表页面？（查看调试信息的URL）
- [ ] 页面是否完全加载？（能看到课程）
- [ ] 网络是否正常？
- [ ] 是否已登录？

**调试信息示例**：
```
📊 调试信息：
• .kbcontent 元素: 0
• Table 元素: 5
• 课表 Table: 0
• 页面状态: complete
• 页面标题: 我的课表

⚠️ 页面内容为空！
💡 建议方案：
1. 使用JSON导入方式（更可靠）
2. 或等待页面完全加载后重试
```

## 🎓 技术说明

### 为什么需要等待？

动态页面加载过程：
```
1. HTML加载   [====>    ] 1秒
2. JavaScript [====>    ] 1秒  
3. AJAX请求   [======>  ] 2秒
4. DOM渲染    [========>] 1秒
   ↓
共需约5秒完全加载
```

应用的处理：
- ⏳ 初始等待：2秒
- 🔄 检测到空页面：再等3秒
- 🔄 最多重试：3次
- ⏱️ 总等待时间：最长11秒

### 提取优先级

```javascript
1. .kbcontent     // 优先级最高
   ↓ 失败
2. table[id*="kb"] // 次优先
   ↓ 失败  
3. window[var]     // 全局变量
   ↓ 失败
4. 提示用户使用JSON导入
```
