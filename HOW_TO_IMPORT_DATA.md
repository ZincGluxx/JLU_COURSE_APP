# 📚 课表数据导入完整指南

## 问题说明

如果应用提示"接口无法获取数据"或其他错误，可以使用本指南手动从网页导入课表数据。

---

## 📱 方法一：使用应用内置浏览器（推荐）

### 步骤详解

#### 1. 打开导入功能

1. 启动应用
2. 点击底部的 **"设置"** 标签
3. 找到 **"重新获取课表"** 选项
4. 点击进入WebView浏览器

#### 2. 登录教务系统

WebView会自动打开 `https://i.jlu.edu.cn`

1. 使用您的学号和密码登录
2. 完成任何二次验证（如果有）
3. 等待登录成功

#### 3. 找到课表页面

登录成功后，有几种方式找到课表：

**方式A：通过首页导航**
- 在首页找到 **"我的课表"** 或 **"课程表"** 链接
- 点击进入

**方式B：直接访问**
- 如果知道课表URL，可以在地址栏直接输入
- 常见URL：
  - `https://iedu.jlu.edu.cn/jwapp/sys/wdkb/*`
  - `https://i.jlu.edu.cn/jwapp/sys/*/index.do`

#### 4. 等待页面加载

- 确保课表完全显示在页面上
- 可以看到所有课程信息
- 等待页面停止加载（顶部进度条消失）

#### 5. 点击提取按钮

在WebView页面的**右上角**，点击：
- 📥 **下载图标**（向下箭头）

或者当页面检测到课表时，会自动弹出提示：
- ✅ **检测到课表页面，可以提取数据**
- 点击提示中的 **"提取"** 按钮

#### 6. 查看提取结果

提取成功后：
- ✅ 显示 "成功提取 X 门课程"
- 自动返回主页
- 课程表中显示导入的数据

## 🔧 方法二：调试模式（如果方法一失败）

### 步骤1：查看页面HTML结构

如果自动提取失败，需要手动查看HTML结构：

#### 在应用中查看

1. 进入WebView浏览器
2. 导航到课表页面
3. 点击右上角下载按钮
4. 如果提取失败，会显示错误对话框
5. 错误对话框中可能包含部分HTML内容

#### 使用Chrome DevTools（更详细）

如果需要更详细的调试：

1. 在电脑浏览器打开 `https://i.jlu.edu.cn`
2. 登录并进入课表页面
3. 按 `F12` 打开开发者工具
4. 切换到 **"Elements"** 或 **"元素"** 标签
5. 找到课表区域的HTML结构

### 步骤2：查找课表元素

在HTML中查找课程信息的特征：

#### 常见的HTML模式

**模式1：div.kbcontent**
```html
<div class="kbcontent">
    高等数学<br>
    张三<br>
    1-16周<br>
    第1-2节<br>
    教学楼A101
</div>
```

**模式2：table结构**
```html
<table id="kbtable">
  <tr>
    <td>高等数学 张三 教A101</td>
  </tr>
</table>
```

**模式3：data属性**
```html
<td data-weekday="1" data-section="1">
    课程信息...
</td>
```

### 步骤3：记录课表数据特征

需要记录以下信息：
- ✅ 课程名称在哪里？
- ✅ 教师名称在哪里？
- ✅ 上课地点在哪里？
- ✅ 周次信息格式（如 "1-16周"）
- ✅ 节次信息格式（如 "1-2节"）
- ✅ 星期信息如何表示？

### 步骤4：手动构建课程数据

如果自动提取一直失败，可以手动创建课程数据：

#### 方法：通过控制台注入

1. 在WebView中，课表页面加载完成后
2. 开发者可以运行自定义JavaScript代码

## 🐛 常见问题解决

### Q1: 点击下载按钮没反应

**可能原因：**
- 页面未完全加载
- 不在课表页面

**解决方案：**
1. 等待页面完全加载
2. 刷新页面（点击刷新按钮 🔄）
3. 确认URL中包含 "kb" 或 "课表"
4. 重新导航到课表页面

### Q2: 提示"未找到课表数据"

**可能原因：**
- HTML结构与预期不符
- 课表为空（还未选课）
- 页面是动态加载的

**解决方案：**
1. 确认浏览器中能看到课程
2. 等待所有课程加载完成
3. 尝试切换课表视图（周视图/学期视图）
4. 查看Q5了解如何自定义提取

### Q3: 提取的课程数量不对

**可能原因：**
- 部分课程未被识别
- 重复的课程

**解决方案：**
1. 在设置中"清除所有数据"
2. 重新导入
3. 对比网页和应用中的课程数量

### Q4: 课程信息不完整（缺少教师或地点）

**可能原因：**
- HTML结构中信息分散
- 提取逻辑需要优化

**解决方案：**
- 记录缺失的信息
- 导入后可以手动补充（未来版本功能）

### Q5: 需要自定义提取逻辑

如果您熟悉JavaScript，可以：

1. 查看 `lib/widgets/simple_webview_login.dart`
2. 找到 `_extractCourseTable()` 方法
3. 修改JavaScript代码适应您的页面结构

**需要修改的部分：**
```javascript
// 查找课程元素的选择器
var kbcells = document.querySelectorAll('.kbcontent');

// 解析文本的逻辑
var lines = text.split('\\n')...
```

## 📋 数据格式说明

导入的每门课程需要包含：

| 字段 | 说明 | 示例 | 必需 |
|------|------|------|------|
| 课程名称 | 课程全称 | "高等数学A(1)" | ✅ 是 |
| 教师 | 任课教师 | "张三" | ❌ 否 |
| 地点 | 上课地点 | "中心校区-教A101" | ❌ 否 |
| 星期 | 1-7 (周一到周日) | 1 | ✅ 是 |
| 起始节次 | 1-12 | 1 | ✅ 是 |
| 结束节次 | 1-12 | 2 | ✅ 是 |
| 周次列表 | 哪些周上课 | [1,2,3,...,16] | ✅ 是 |

## 🎯 最佳实践建议

### ✅ 推荐做法

1. **首次使用**：完整阅读本指南
2. **选课后**：立即导入最新课表
3. **定期更新**：每学期初更新一次
4. **备份数据**：截图保存导入成功的提示
5. **验证数据**：导入后检查课程数量和信息

### ❌ 避免操作

1. 不要在页面加载中途点击提取
2. 不要在非课表页面尝试提取
3. 不要频繁重复导入（会覆盖数据）
4. 退出前确保已返回主页（数据已保存）

## 🔄 数据更新流程

### 学期初更新课表

1. 打开应用 → 设置
2. 点击"清除所有数据"
3. 点击"重新获取课表"
4. 按照方法一重新导入

### 课程调整后更新

如果有课程变动：
1. 清除旧数据
2. 重新导入全部课表
（注意：当前版本不支持增量更新）
