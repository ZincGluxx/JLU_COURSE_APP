# WebView登录使用指南

## 概述

本应用已实现 **WebView CAS登录** 功能，无需手动输入账号密码，直接在应用内完成吉林大学统一身份认证。

## 使用步骤

### 1. 启动应用

运行应用后，如果未登录会自动显示登录页面。

### 2. 点击"使用网页登录"

在登录页面底部，点击 **"使用网页登录"** 按钮。

### 3. 完成CAS认证

- 应用会加载吉林大学CAS登录页面
- 输入您的学号和密码
- 如需输入验证码，按页面提示完成
- 点击"登录"按钮

### 4. 自动提取Cookie

- 登录成功后，页面会自动跳转到教务系统
- 应用会自动提取登录凭证（Cookie）
- 提示"登录成功！"后自动返回主页

### 5. 开始使用

登录完成后，可以：
- 查看课程表
- 查询成绩
- 所有功能将使用您的真实数据

## 技术细节

### Cookie存储

- Cookie自动存储在本地
- 有效期：**24小时**
- 过期后需要重新登录

### 登录流程

```
用户点击登录
    ↓
加载 CAS 登录页
    ↓
用户输入账号密码
    ↓
CAS 验证成功
    ↓
重定向到教务系统
    ↓
提取 Cookie
    ↓
保存到本地
    ↓
登录完成
```

### 安全性

1. **无密码存储**: 应用不存储您的密码
2. **本地Cookie**: Cookie仅保存在您的设备上
3. **加密传输**: 全程使用HTTPS加密通信
4. **自动过期**: Cookie 24小时后自动失效

## 常见问题

### Q1: 登录页面空白

**原因**: 网络问题或CAS服务器维护

**解决方案**:
1. 检查网络连接
2. 点击刷新按钮（右上角）
3. 等待片刻后重试

### Q2: 登录后显示"未获取到会话信息"

**原因**: Cookie提取失败

**解决方案**:
1. 确认已完全登录到教务系统
2. 重新登录
3. 检查是否被防火墙拦截

### Q3: Cookie过期怎么办？

**现象**: 查询数据时提示"未登录"

**解决方案**:
1. 重新点击"使用网页登录"
2. 应用会自动恢复您的登录状态

### Q4: 能否记住密码？

目前不支持。基于安全考虑，我们不存储密码。每次登录需要在CAS页面输入，但Cookie可保持24小时免登录。

### Q5: 为什么账号密码登录按钮显示"已弃用"？

**原因**: 
- CAS登录需要RSA加密、lt参数提取等复杂流程
- WebView登录更简单、更安全、更可靠

**建议**: 使用WebView登录

## 与旧方法对比

| 特性 | 账号密码登录 | WebView登录 |
|------|------------|-------------|
| 实现难度 | 复杂（RSA加密） | 简单 |
| 安全性 | 需要存储密码 | 不存储密码 |
| 验证码 | 难以处理 | 原生支持 |
| 维护成本 | 高（CAS变化需更新） | 低 |
| 推荐度 | ❌ 不推荐 | ✅ 推荐 |

## 开发者信息

### 相关文件

- `lib/widgets/webview_login.dart` - WebView登录组件

### 关键方法

```dart
// 设置WebView提取的Cookie
apiService.setLoginCookies(cookies, username: '学号');

// 从缓存恢复登录
await apiService.restoreLoginFromCache();

// 检查登录状态
bool isLoggedIn = apiService.isLoggedIn;
```
