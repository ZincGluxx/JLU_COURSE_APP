# WebView 网页登录使用指南

## 功能说明

WebView 登录方案允许你直接在应用内通过网页登录教务系统，无需配置复杂的 API 接口。应用会自动尝试从网页中提取课程表数据。

## 使用步骤

### 方法一：自动提取（推荐）

1. **打开应用** → 进入"设置"页面 → 点击"未登录"

2. **点击"使用网页登录"按钮**
   - 应用会打开内置浏览器
   - 自动跳转到吉林大学教务系统登录页面

3. **在网页中登录**
   - 输入学号和密码
   - 完成登录（包括验证码等）

4. **进入课程表页面**
   - 登录成功后，导航到课程表页面
   - 应用会自动检测并尝试提取数据

5. **使用提取的数据**
   - 如果检测到课程数据，会弹出确认对话框
   - 点击"使用此数据"保存到应用

### 方法二：手动提取

如果自动提取没有成功：

1. **登录后手动导航到课程表页面**
   - 确保完全加载课程表

2. **点击右上角的下载图标**（↓）
   - 手动触发数据提取

3. **查看提取的数据**
   - 在弹出的对话框中查看数据格式
   - 如果数据正确，点击"使用此数据"

### 方法三：查看 Cookie（高级）

如果需要手动配置 API：

1. **登录教务系统**

2. **点击右上角的 Cookie 图标**（🍪）
   - 查看当前的 Cookie 信息

3. **复制 Cookie**
   - 在弹出的对话框中复制完整的 Cookie
   - 可用于配置 API 服务

## 工具栏说明

右上角提供了以下工具：

- **🔄 刷新** - 重新加载当前页面
- **↓ 下载** - 手动提取课程数据
- **🍪 Cookie** - 查看 Cookie 信息
- **⋮ 更多**
  - 后退 - 返回上一页
  - 前进 - 前往下一页
  - 跳转到课程表 - 直接打开课程表页面

## 支持的数据提取方式

应用使用以下方法尝试提取数据：

### 1. JavaScript 全局变量
如果网页将课程数据存储在全局变量中（如 `courseData`），应用会自动读取。

### 2. DOM 元素解析
从页面的 HTML 元素中提取课程信息：
- 课程名称
- 授课教师
- 上课地点
- 上课时间

### 3. 页面内容分析
检测页面是否包含课程相关内容。

## 常见问题

### Q1: 为什么没有自动提取数据？

**可能原因**：
- 页面还在加载中
- 课程表数据使用了特殊的格式
- 页面使用了动态加载（AJAX）

**解决方案**：
1. 等待页面完全加载
2. 使用"手动提取"功能
3. 多次尝试提取

### Q2: 提取的数据不完整或不正确？

**解决方案**：
1. 确保已经完全加载课程表页面
2. 检查页面上是否显示了所有课程
3. 尝试切换到不同的周次或学期
4. 使用浏览器 F12 查看数据实际格式

### Q3: 登录后无法进入课程表页面？

**解决方案**：
1. 点击"更多" → "跳转到课程表"
2. 或在网页中手动导航
3. 确认账号有权限访问课程表

### Q4: 验证码问题？

WebView 登录的**优势**就是支持验证码！
- 可以像在浏览器中一样手动输入验证码
- 支持图片验证码、滑块验证码等

## 隐私和安全

- **数据本地存储**：提取的数据仅存储在你的设备上
- **无服务器传输**：不会将你的密码或数据发送到第三方服务器
- **Cookie 隔离**：WebView 的 Cookie 独立于系统浏览器
- **HTTPS 加密**：所有网络通信使用 HTTPS 加密

## 故障排查

### 4. 查看日志

## 技术细节

### JavaScript 注入
应用通过 `runJavaScriptReturningResult` 在网页中执行 JavaScript 代码来提取数据。

### Cookie 管理
使用 WebView 的内置 Cookie 管理，自动处理登录状态。

### 跨域限制
由于代码在网页的上下文中运行，不受跨域限制。
