# 🚀 课表导入快速指南

## 第一步：启动导入流程

1. 打开应用 → 底部 **"设置"** 
2. 点击 **"重新获取课表"**
3. 进入WebView浏览器

## 第二步：登录并找到课表

### 登录
- 浏览器自动打开 `https://i.jlu.edu.cn`
- 输入学号、密码登录
- 完成验证

### 找到课表页面
在首页找到并点击：
- **"我的课表"** 
- **"课程表"**
- **"教学管理"** → **"我的课表"**

## 第三步：使用调试工具（新增）

### 🐛 查看页面信息
点击右上角 **虫子图标** (🐞)，会显示：

**如果看到的数字：**
- `.kbcontent: 0 个` → 不在课表页面或HTML结构不同
- `.kbcontent: > 0 个` → ✅ 可以提取！
- `table[id*="kb"]: > 0 个` → ✅ 可以提取！

**示例内容区域**：
- 如果显示课程名称、教师名 → ✅ 正确页面
- 如果是其他内容 → ❌ 需要继续导航

## 第四步：提取数据

### 方式A：自动提取
- 页面加载后自动检测
- 底部弹出提示："检测到课表页面"
- 点击提示中的 **"提取"** 按钮

### 方式B：手动提取  
- 点击右上角 **下载图标** (📥)
- 等待提取完成

## 如果提取失败

### 1. 查看错误信息
错误对话框会显示：
```
📊 调试信息：
• .kbcontent 元素: 0
• Table 元素: 5
• 课表 Table: 0
• 页面标题: 吉林大学教务管理系统
• 当前URL: https://...
```

### 2. 分析问题

**情况1：所有元素都是0**
→ 不在课表页面，继续导航

**情况2：有table但找不到课表table**
→ 网页结构变化，需要调整代码

**情况3：有.kbcontent但提取失败**
→ 页面未完全加载，刷新后重试

### 3. 使用调试按钮深入检查

点击 🐞 按钮查看：
- 页面标题和URL
- 页面文本预览（确认是否有课程信息）
- 示例内容（查看第一个课程的原始文本）

## 成功标志

✅ 提示："成功提取 X 门课程"
✅ 自动返回主页
✅ 课程表中显示课程

## 常见问题速查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 点击下载无反应 | 页面未加载完 | 等待或刷新 |
| 提取0门课程 | 不在课表页面 | 检查URL和页面文本 |
| 课程数量不对 | 部分未识别 | 查看调试信息中的示例内容 |
| 提示找不到元素 | 网页结构变化 | 需要更新代码（见下方） |

## 给开发者：如何更新提取代码

如果调试信息显示页面有课表，但代码无法提取：

### 1. 记录HTML结构
使用电脑浏览器：
1. 打开 `https://i.jlu.edu.cn`
2. 登录并进入课表页面
3. 按 `F12` → Elements 标签
4. 找到课表区域的HTML
5. 记录：
   - 课程信息在什么元素里？（class/id）
   - 每个字段（名称、教师、地点）的位置？
   - 星期和节次如何表示？

### 2. 修改提取代码
编辑文件: [lib/widgets/simple_webview_login.dart](lib/widgets/simple_webview_login.dart)

找到 `_extractCourseTable()` 方法（约110行）

修改JavaScript选择器：
```javascript
// 当前代码：
var kbcells = document.querySelectorAll('.kbcontent');

// 如果你的页面用不同的class：
var kbcells = document.querySelectorAll('.course-cell');
// 或
var kbcells = document.querySelectorAll('.kb-item');
```

### 3. 修改文本解析
如果字段顺序不同，调整解析逻辑：
```javascript
// 当前假设：
// lines[0] = 课程名
// lines[1] = 教师
// lines[2] = 地点
// lines[3+] = 周次/节次

// 根据实际情况调整
```
